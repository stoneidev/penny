## 🔍 이전 보고서와의 차이 및 왜곡 원인 분석

기존 시뮬레이션의 누적 수익률(+254% / +831%)이 실제 성과(+53% / +89%)와 큰 괴리를 보였던 핵심 원인은 다음과 같습니다.

### 1. 제한된 유니버스로 인한 왜곡 (Universe Restriction)
기존 코드는 전체 미국 시장 갭상승 2위가 아니라, 미리 수집된 **650개 종목 풀 안에서만 갭 2위**를 선정했습니다. 그 결과 진짜 갭 2위 종목(예: 5/1 `ESPR` +56.5%, 5/4 `GBTG` +56.9%, 5/5 `CLRB` +59.4% 등)이 풀에 존재하지 않아 매수 대상에서 대거 누락되었습니다.

### 2. 분봉 데이터의 부재 및 날짜 버그
기존 로컬 JSON 캐시에 1분봉 데이터가 없어 실제로 일봉 폴백 데이터를 사용한 일자가 존재하였으며, 인덱스 시간 계산 버그(RangeIndex 파싱 오류)로 인해 오전 10시 타임컷이 완전 무효화되어 장중 고점 도달이 과다 반영된 점이 수익률을 극대화하여 부풀렸습니다.

본 버전에서는 전체 시장 스크리닝을 진행하고 yfinance API의 실시간 1분봉/5분봉을 다운로드하여 정직하게 청산을 처리함으로써 실거래 승률 **42.3%**, 누적 수익률 **+53.63% (50% 베팅)**의 정상 수치로 확인되었습니다.

## 🛠️ 전략 매매 규칙 (Strategy Specifications)

1. **대상 유니버스**: NASDAQ, NYSE, AMEX 상장 보통주 (Common Stock) 5,197개
   * ETF, 우선주, 워런트, SPAC, REITs 등 파생/우선 거래 자산 완전 제외
2. **진입 조건**: 매일 아침 09:30 시초가 기준, 갭상승률 **전체 미국 시장 2위 종목** 매수
   * *1위 제외 이유*: 거래정지(Halt) 리스크, 인수합병(M&A) 매수 호가 고정, 과도한 변동성을 회피하기 위함
3. **진입 필터**: 시초가 가격 \$1.00 ~ \$10.00, 전일 거래대금(Turnover) $\ge$ \$50,000
   * 2위 종목의 시초가 갭상승률이 **+200% 이상일 경우 거래 미진입 (No Trade)**
4. **익절/손절**: 진입가 대비 **+50.0% Take Profit (TP)** / **-20.0% Stop Loss (SL)**
5. **강제 타임컷**: **10:00 AM EST** (장 개시 후 30분) 시점까지 미청산 시 당시 가격(분봉 close)으로 즉시 시장가 강제 청산

## 📂 프로젝트 구조

```
penny/
├── README.md                 # 본 문서 (전략 리포트)
├── data/
│   ├── universe_v2.txt       # 필터링된 5,197개 보통주 티커 목록
│   ├── picks_v2.csv          # 일자별 갭상승 2위 종목 스크리닝 결과
│   ├── trades_v2.csv         # 각 거래의 상세 진입/청산/수익률 로그
│   └── equity_v2.csv         # 날짜별 누적 자본금(100% 및 50% 베팅) 추이
└── src/
    ├── v2_01_universe.py     # 미국 3대 거래소 보통주 유니버스 추출 스크립트
    ├── v2_02_picks.py        # 유니버스 일봉 데이터 다운로드 및 매일 갭 2위 종목 선정
    ├── v2_03_simulate.py     # 분봉 기반의 정밀 매매 시뮬레이션 실행 (TP/SL/타임컷)
    └── v2_04_report.py       # 시뮬레이션 최종 수익률 계산 및 리포트 출력
```

## 🚀 실행 방법 (How to Run)

프로젝트 루트 디렉토리에서 아래 순서대로 스크립트를 실행하면 백테스트를 처음부터 직접 재현할 수 있습니다.

```bash
# 1. 미국 보통주 유니버스 파일 생성 (data/universe_v2.txt)
python3 src/v2_01_universe.py

# 2. 일봉 데이터를 받아 매일의 갭상승 2위 종목 선정 (data/picks_v2.csv)
python3 src/v2_02_picks.py

# 3. 1분봉/5분봉을 내려받아 TP/SL/타임컷 백테스트 진행 (data/trades_v2.csv)
python3 src/v2_03_simulate.py

# 4. 누적 수익률 계산 및 분석 리포트 화면 출력 (data/equity_v2.csv)
python3 src/v2_04_report.py
```

## ⚠️ 유의 사항 및 한계점
* **통계적 유의성**: 백테스팅 기간이 30거래일(실거래 26건)로 짧아 통계적 표본이 적으며, 성과 대부분이 3건의 익절(TP) 거래에 집중되어 있습니다.
* **비용 미반영**: 슬리피지(Slippage) 및 거래 수수료, 호가 스프레드가 미반영된 수치입니다. 실제 소형 페니주의 경우 시초가 시장가 매수 및 타임컷 강제 청산 시 수 퍼센트의 슬리피지가 발생할 수 있어 실제 순수익률은 본 결과보다 낮아질 가능성이 큽니다.
