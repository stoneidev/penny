# 📊 미국 페니주 갭상승 매매 시스템 백테스팅 및 전략 검증

이 리포지토리는 미국 소형주(페니 스톡)의 시초가 갭상승 매매 전략을 백테스팅하고 검증하기 위한 코드 및 시뮬레이션 결과 데이터를 포함하고 있습니다. 

본 프로젝트에서는 **2026년 5월 1일부터 6월 12일까지(총 30개 거래일)**의 역사적 1분봉 데이터 및 일봉 데이터를 기반으로 다양한 익절/손절/청산 규칙을 테스트하였습니다.

---

## 💡 주요 발견: 인덱스 시간 버그 (RangeIndex Bug) 및 올바른 수익률 검증

초기 시뮬레이션 코드(`test_pt50.py`, `print_pt50_details.py`)에서 1분봉 데이터를 로드한 뒤 인덱스를 재정의(`df_1m.sort_values("timestamp").reset_index(drop=True)`)하는 과정에서 데이터프레임의 인덱스가 **날짜(DatetimeIndex)가 아닌 정수형(RangeIndex: 0, 1, 2, ...)**으로 초기화되는 현상이 발생했습니다.

### 1. 버그의 영향
* 이 상태에서 코드 내 `df_1m.index[i]`를 통해 시간을 추출하고 `pd.to_datetime()`으로 변환하면서, 모든 분봉의 시간이 **`1970-01-01 00:00:00` 근처**로 파싱되었습니다.
* 그 결과, **오전 10:00 강제 청산(Time Stop) 조건(`current_time > time_limit`)이 단 한 번도 작동하지 않는 심각한 오류**가 발생했습니다.
* 즉, 오전 10:00에 강제 청산되어야 했던 종목들이 장중(낮 12시, 오후 2시 등)에 뒤늦게 익절 기준(+50%)을 터치했음에도 아침에 익절 성공한 것으로 낙관적으로 처리되었습니다.
* 이 오류로 인해 성공 횟수가 비정상적으로 부풀려져 초기 시뮬레이션에서는 **최종 자금 930.82원 (+830.82% 수익률)**이라는 비현실적인 결과가 출력되었습니다.

### 2. 버그 수정 및 최종 검증 결과
* 데이터프레임의 인덱스가 아닌 실제 컬럼값(`df_1m.iloc[i]['timestamp']`)을 참조하여 정확한 시간을 체크하도록 코드를 수정하였습니다.
* 버그를 수정하여 **10:00 AM 타임컷(30분 강제 시장가 청산)**이 정상 작동하도록 한 뒤 재시뮬레이션한 최종 검증 결과는 아래와 같습니다:
  * **전액 복리(All-in) 최종 자금**: **635.82원 (+535.82% 수익률)** (MDD: 40.25%)
  * **50% 투자 + 50% 현금 예치(50% 베팅) 최종 자금**: **287.53원 (+187.53% 수익률)** (MDD: 21.94%)
  * **실제 승률**: **63.3%** (30일의 시뮬레이션 기간 중 23일 진입, 7일은 필터 스킵. 진입일 중 승률 기준)
  * **평균 수익률**: 거래당 **+8.10%**

## 🛠️ 전략 규칙 (Strategy Rules)

검증을 거쳐 확정된 매매 전략의 정확한 규칙은 다음과 같습니다:

1. **대상 유니버스**: 거래대금과 가격 필터를 통과한 **650개 소형주 유니버스** (`/price_cache` 내 CSV 파일 목록)
2. **대상 종목 선정**: 매일 아침 09:30 장 개시 직전, 갭상승률 **전체 2위 종목** 진입
   * *1위 종목을 제외(Skip #1)하는 이유*: 1위 종목은 종종 인수합병(M&A), 상장폐지 절차 등으로 인해 거래 정지(Halt)되거나 갭이 너무 높게 떠서 하방 압력이 비정상적으로 강하기 때문에 안정적인 2위를 공략합니다. (예: 5월 7일 1위 `RXT`가 +75.1% 갭상승했으나 필터로 제외하고 2위 `ARTR` 진입)
3. **진입 필터**: 시초가 갭상승률이 **+200% 미만**일 때만 매수 진입 (200% 이상일 경우 당일 거래 없음)
4. **진입 시점**: 장 개시 시초가(09:30 AM EST)에 시장가 매수
5. **익절 기준 (PT - Profit Target)**: 진입가 대비 **+50.0%** (또는 +40.0% 대안) 터치 시 즉시 지정가 매도 청산
6. **손절 기준 (SL - Stop Loss)**: 진입가 대비 **-20.0%** 터치 시 즉시 손절 청산 (장 초반 Wiggle을 견디기 위한 마지노선)
7. **강제 시간 청산 (Time Stop)**: **오전 10:00 AM EST** (장 개시 후 30분 경과)까지 익절/손절이 모두 체결되지 않은 경우, 당시 가격으로 시장가 **강제 청산**

### 📈 핵심 인사이트
1. **고정 익절(Fixed PT)의 우위**: 페니주는 아침 개장 직후 3~10분 사이에 강한 Morning Spike(급등)를 준 뒤 수직 낙하(Dump)하는 경우가 대다수입니다. 따라서 고점 대비 하락을 기다리는 **트레일링 스탑은 급격한 되돌림으로 인해 대부분의 이익을 반납**하게 되며, 최고점에 일시 도달했을 때 즉시 이익을 확정 짓는 **고정 익절이 훨씬 높은 수익률을 기록**합니다.
2. **10:00 AM 타임컷의 중요성**: 타임컷 없이 장 마감(EOD Close)까지 포지션을 보유하면 장중 상승분이 모두 소멸하여 누적 손실이 매우 커집니다. 장 초반 30분의 변동성 모멘텀만 빠르게 취하는 것이 핵심 생존 법칙입니다.
3. **자금 관리 (50% 베팅)**: All-in 복리 투자는 폭발적인 수익을 내지만 **최대 낙폭(MDD)이 40.25%**에 달합니다. 자산의 50%만 베팅하고 50%는 현금으로 유지하는 전략은 **MDD를 21.94%로 절반 가까이 낮추면서도 +187.53%의 우수한 복리 수익**을 냅니다.

## 🚀 실행 방법 (How to Run)

### 1. 환경 변수 설정
백테스팅 실행 시 로컬의 폴리곤 분봉 캐시 경로 및 가격 데이터를 참조합니다. 환경 변수를 통해 경로를 커스텀할 수 있습니다. (설정하지 않을 경우 디폴트 경로로 설정됩니다.)

```bash
export PRICE_CACHE_DIR="/path/to/price_cache"
export MINUTE_CACHE_DIR="/path/to/polygon_minute_cache"
```

### 2. 종합 시뮬레이션 실행
종합적인 매개변수 조합(고정 익절 vs 트레일링 스탑, 타임컷 유무 등)을 시뮬레이션하고 비교표를 생성합니다.

```bash
python3 src/run_comprehensive.py
```

### 3. PT +50% 상세 흐름 리포트 생성
가장 우수한 성과를 보인 PT +50% / SL -20% / 10:00 AM 타임컷 전략의 일자별 상세 거래 이력을 출력합니다.

```bash
python3 src/print_pt50_details.py
```
